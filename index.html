    Риск аппаратного сбоя

    Из-за износа оборудования серверы становятся со временем уже не такими надёжными и
    отказоустойчивыми, какими они были при покупке.
    В итоге, его вычислительной мощности не хватает (по быстродействию процессора, объёму оперативной
    памяти, дисковой ёмкости, пропускной способности и т.п.) и возникают перебои/зависания в работе всей
    ИТ-инфраструктуры организации.
    Такие перебои или, ещё хуже, простои опять ведут к материальным убыткам и к потерям репутации.

    Риск безопасности ОС

    Это означает, что любые новые уязвимости, обнаруживаемые в Windows Server 2008/2008R2 после
    окончания поддержки, не будут исправляться в актуальных обновлениях безопасности Майкрософт.
    В чем же заключаются риски при работе на Windows Server 2008/2008R2 после завершения поддержки?
    Один из них состоит в том, что в борьбе за безопасность сервера с Windows Server 2008/ 2008R2 у
    нападающих будет преимущество перед защитниками: злоумышленники, весьма вероятно, будут больше знать об уязвимостях этой операционной системы

    Гиперконвергентная инфраструктура

    -Служба миграции хранилища

    Новая технология, упрощающая миграцию служб хранилища
    на более новую версию Windows Server.

    Управление SMS осуществляется через Центр администрирования Windows, где
    он предоставляет графические мастера, которые проводят инвентаризацию
    данных на серверах, переносят данные и конфигурацию на более новые серверы,
    а затем при необходимости переносят идентификационные данные старых
    серверов на новые серверы. Большим преимуществом SMS является то, что
    пользователям не нужно меняться после миграции. Поскольку имя сервера,
    IP-адрес, общие ресурсы и разрешения сохраняются, нет необходимости
    настраивать сценарии входа в систему или сопоставления сетевых дисков.

    Моменты которые следует учитыват при миграции файловых серверов:

    Определить файлы, которые необходимо перенести, а
    также организовать и хранить.

    Выполнить миграцию, включая настройки
    разрешений для папок / файлов.

    Копирование данных занимает гораздо больше
    времени, чем ожидается.

    Копирование может быть не завершено за выходные
    дни, что может повлиять на доступность данных.

    Вы должны сделать резервную копию своих данных
    перед миграцией на случай потери данных.
